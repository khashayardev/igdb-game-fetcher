# 🎮 IGDB Complete Game Scraper

یک اسکریپت حرفه‌ای پایتون برای دریافت و ذخیره‌سازی کامل دیتابیس بازی‌ها از API رسمی IGDB (Twitch). این ابزار با استفاده از GitHub Actions به صورت خودکار اجرا شده و اطلاعات جامعی شامل ژانر، پلتفرم، تصاویر، ویدیوها، شرکت‌های سازنده، امتیازات و رده‌بندی سنی را برای تمام بازی‌های منتشر شده بین سال‌های ۲۰۰۰ تا ۲۰۲۶ دریافت می‌کند. برخلاف اسکریپت‌های ساده، این نسخه با مدیریت هوشمند pagination، محدودیت ۵۰۰ تایی API را دور زده و **تمام بازی‌های هر سال** (به عنوان مثال بیش از ۲۳۰۰ بازی برای سال ۲۰۰۰) را بدون نقص دریافت می‌کند.


## 🚀 نصب و اجرا

برای استفاده کافیست مخزن را Fork کرده، در بخش `Settings → Secrets and variables → Actions` دو Secret به نام‌های `TWITCH_CLIENT_ID` و `TWITCH_CLIENT_SECRET` (دریافت از [Twitch Developer Portal](https://dev.twitch.tv/console)) تنظیم کنید. سپس در تب Actions، workflow مورد نظر را انتخاب و با تعیین بازه سال (مثلاً 2000 تا 2026) و حداقل امتیاز دلخواه (پیش‌فرض 0)، اجرا کنید. پس از اتمام (۱۰ تا ۱۵ دقیقه برای بازه کامل)، فایل JSON نهایی از بخش Artifacts قابل دانلود است. برای اجرای محلی نیز کافیست کتابخانه `requests` نصب و متغیرهای محیطی را تنظیم کنید.


## 📊 خروجی و کاربرد

خروجی شامل یک فایل JSON با ساختار منظم شامل metadata (آمار دریافت، تعداد بازی‌ها، تعداد درخواست‌های API) و لیست کامل بازی‌ها به همراه تمام فیلدهای ضروری مانند `name`, `summary`, `first_release_date`, `genres_summary`, `platforms_summary`, `cover_url`, `screenshots_urls`, `videos_summary`, `companies_summary`, `rating`, `age_ratings_summary` و ... می‌باشد. این داده‌ها برای ساخت پایگاه داده بازی، وب‌سایت‌های نقد و بررسی، سیستم‌های پیشنهاددهنده، اپلیکیشن‌های موبایل و هر پروژه دیگری که نیاز به دیتای کامل و ساختاریافته بازی‌ها دارد، ایده‌آل است. تحت مجوز MIT منتشر شده و آزادانه قابل استفاده است.

## 🔧 سفارشی‌سازی و توسعه

اسکریپت کاملاً ماژولار طراحی شده و به راحتی قابل توسعه است. می‌توانید با تغییر لیست `self.fields` در کلاس اصلی، فیلدهای دلخواه خود را اضافه یا حذف کنید. همچنین قابلیت فیلتر بر اساس پلتفرم خاص (مثلاً فقط بازی‌های PC با کد 6) یا محدوده امتیاز مشخص، به راحتی با تغییر شرط `where` در کوئری‌ها امکان‌پذیر است. برای بهینه‌سازی سرعت، پارامترهای `REQUEST_DELAY` و `CHUNK_DELAY` قابل تنظیم هستند و می‌توانید با تغییر `YEAR_CHUNK_SIZE`، تعداد سال‌های پردازش همزمان را کنترل کنید.
